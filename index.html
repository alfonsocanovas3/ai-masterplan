🏗️ Platform Strategy: Cowork + Neural Box Convergence

Dolead has two AI platforms that need to converge, not compete. The recommendation: Neural Box becomes the backbone (Slack-accessible for all 69 employees), Cowork stays as the power-user layer for anyone who needs deep browser automation.

Recommended Architecture

Neural Box = V1 backbone. All 31 agents run here. Every Doleader accesses via Slack (@dolead-ai). Magnus routes requests to the right skill. Scheduled tasks (morning briefs, campaign monitoring) run automatically. This covers 90% of use cases.

Cowork = Power-user layer. For people who need live browser control, complex document creation, or deep interactive debugging (Alfonso, Arthur, and whoever needs it). Cowork plugins can call Neural Box APIs for data, and Neural Box skills can be tested/prototyped in Cowork first.

📝 Vlad Meeting Agenda (10 Items)

Before building, Alfonso and Vlad need to align on these 10 topics. This is the agenda for the convergence meeting.

1. Magnus Architecture
How does Magnus route requests? Static mapping (keyword to skill) or LLM-based intent recognition? Where does it live (Neural Box)? How does it handle multi-agent chaining?
2. Skill Format Convergence
Can Neural Box skills and Cowork plugin SKILL.md files share the same format? Should we standardize on one structure so agents are portable between platforms?
3. Cowork-to-Neural-Box Porting
How to port TechOps Expert, Studio Expert knowledge to Neural Box? The knowledge bases (50+ patterns, CRO frameworks) are the real asset. Browser automation needs Playwright adaptation.
4. User Identity & Permissions
When a CSM asks Magnus something, how does Magnus know their squad, clients, permissions? Does Neural Box have a user profile system? How do we map Slack users to Dolead roles?
5. Shared Tool Layer
Both platforms use MCP. Can we share the tool definitions (ClickUp, Slack, Looker, Admin, Ops)? Neural Box has 112 tools, some overlap with Cowork connectors.
6. Ad Platform APIs
TrafficOptimizer agents need Google Ads, Meta Ads, TikTok Ads API access. Does Neural Box already have these? What auth flow? Rate limits? Read-only first vs read-write?
7. Monitoring & Alerting Infrastructure
Campaign-Watch, Quality-Monitor, LDR-Troubleshooter all need to run on schedules and push alerts to Slack. Does Neural Box's scheduler support this pattern? What's the latency?
8. Knowledge Base Architecture
Where do agent knowledge bases live? Cowork uses reference .md files that grow over time. Neural Box uses skills repo. How do we keep knowledge in sync across platforms?
9. Cost Model
69 users x multiple daily interactions = significant API costs. What's Neural Box's current cost per interaction? Do we need usage tiers? Rate limiting? Model selection per agent (Haiku for simple, Opus for complex)?
10. Phase 1 Scope Agreement
Agree on which 8 agents go live first, who owns building each one (Alfonso vs Vlad vs shared), and the timeline. Target: 4 weeks to first Slack-accessible agents.
