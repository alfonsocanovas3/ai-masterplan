Magnus Architecture

How does Magnus route requests? Static mapping (keyword to skill) or LLM-based intent recognition? Where does it live (Neural Box)? How does it handle multi-agent chaining?

Skill Format Convergence

Cowork-to-Neural-Box Porting

How to port TechOps Expert, Studio Expert knowledge to Neural Box? The knowledge bases (50+ patterns, CRO frameworks) are the real asset. Browser automation needs Playwright adaptation.

Platform
04

User Identity & Permissions

When a CSM asks Magnus something, how does Magnus know their squad, clients, permissions? Does Neural Box have a user profile system? How do we map Slack users to Dolead roles?

Architecture
05

Shared Tool Layer

Both platforms use MCP. Can we share the tool definitions (ClickUp, Slack, Looker, Admin, Ops)? Neural Box has 112 tools, some overlap with Cowork connectors.

Architecture
06

Ad Platform APIs

TrafficOptimizer agents need Google Ads, Meta Ads, TikTok Ads API access. Does Neural Box already have these? What auth flow? Rate limits? Read-only first vs read-write?

Platform
07

Monitoring & Alerting Infrastructure

Campaign-Watch, Quality-Monitor, LDR-Troubleshooter all need to run on schedules and push alerts to Slack. Does Neural Box's scheduler support this pattern? What's the latency?

Process
08

Knowledge Base Architecture

Where do agent knowledge bases live? Cowork uses reference .md files that grow over time. Neural Box uses skills repo. How do we keep knowledge in sync across platforms?

Architecture
09

Cost Model

69 users x multiple daily interactions = significant API costs. What's Neural Box's current cost per interaction? Do we need usage tiers? Rate limiting? Model selection per agent (Haiku for simple, Opus for complex)?
